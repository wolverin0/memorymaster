## TOP 10 RISKS

### 1. CRITICAL: Hardcoded path in claim_verifier.py:33
Leaks personal OneDrive path as default. Breaks on any other machine.

### 2. HIGH: 35+ silent except Exception: pass blocks
In mcp_server.py, llm_provider.py, wiki_engine.py, operator_queue.py. If Gemini API key expires, everything silently fails.

### 3. HIGH: Raw sqlite3.connect() in mcp_server.py:280
Bypasses WAL mode, foreign keys, retry logic. Potential DB corruption.

### 4. HIGH: Dashboard has no authentication
13 routes exposed with zero auth.

### 5. HIGH: Webhook has no signature verification
Fires to URL with claim data, no HMAC signing.

### 6. MEDIUM: 7 files exceed 800-line limit
storage.py (2,032), cli.py (1,793), postgres_store.py (1,655), steward.py (1,627), operator.py (1,449), dashboard.py (1,059), mcp_server.py (908).

### 7. MEDIUM: No migration framework
Schema via try/except ALTER TABLE. No version tracking or rollback.

### 8. MEDIUM: 6 failing tests
Dedup and tenant isolation regressions.

### 9. MEDIUM-LOW: Dynamic SQL column names in db_merge.py
From PRAGMA on user-supplied DB file.

### 10. MEDIUM: No LLM budget cap per cycle
200+ LLM calls possible per steward run.
