E · The meta-pattern (the actual answer)

The model-induced gap signature here is not broken code — it's disconnected code. Every unit works; the failure lives in the seams:
  • Tested-but-unexercised: features whose tests construct the world manually (bind a session, set a weight) while no production surface ever does — SessionTracker, W_HOLDER, volunteer_context's gate. The tests pass forever; the feature never runs.
  • Cross-feature seams between separately-built features: checkpoint (batch 1) never learned about holder (batch 2). Per-feature agents + per-feature tests structurally cannot see this.
  • Docs asserting intent, not behavior: "opt-in hook" (nothing installs it), "7-hook stack" (6 registered), frozen badges/handbook — docs written at design time and never re-derived from code.
  • Double-gating: a flag added in one pass, its weight defaulted safe in another — individually correct, jointly a no-op.
Countermeasure that would have caught ~80% of this: a periodic reachability audit (exactly this exercise: trace each flag/tool/field from a real production entry point) + generated-not-hand-typed docs (badges, env-var reference, tool counts). Worth running after every multi-feature batch.
