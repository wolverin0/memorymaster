# R1.4 legacy sensitivity inventory

R1.4 supplies an aggregate-only, read-only dry-run inventory for MM-SEC-03 and MM-SEC-04. It dynamically accounts for SQLite user tables, views, columns, schema definitions, explicit artifact and spool roots, and an injected Qdrant payload reader. It never cleans up, redacts, migrates, initializes, rebuilds, backs up, or contacts an external service.

```text
python scripts/sensitivity_inventory.py --db <DATABASE> --artifact-root <ARTIFACT_ROOT> --spool-root <SPOOL_ROOT> --max-file-bytes <BYTES> --max-entries <COUNT>
```

Output is aggregate JSON only: fixed `mode` (`dry_run`), classification (`LEGACY-SENSITIVITY-INVENTORY`), recommendation (`REVIEW_ONLY`), and a fixed top-level status. SQLite records `tables_accounted`, `columns_accounted`, unique-row `records_scanned`/`records_flagged`, derived table/view/column/record counts, bounded schema-identifier/definition findings, and safe-category surfaces. Each surface contains per-field `records_scanned`, `records_flagged`, and bounded `finding_counts`. Safe fields cover R1.4 claim, citation, event, verbatim, Atlas, cache/miner/rule/entity/link/timeline names; unknown columns are classified only as `field_text`, `field_json`, `field_embedding`, `field_binary`, `field_numeric`, or `field_other`. Artifact/spool records contain fixed status/reason, scanned, sensitive, truncated, unavailable, and refused counts plus fixed `file.metadata`, `file.text`, `file.json`, `file.binary`, or `file.other` surfaces. It contains no persisted values, IDs, schema identifiers, sample content, filenames, supplied paths, offsets, payloads, or exception text.

SQLite uses an explicit `mode=ro` URI followed by `PRAGMA query_only=ON` and one `BEGIN` read snapshot; it does not initialize or migrate. SQLite internals are excluded. A quote-aware SQL tokenizer ignores comments only outside quoted identifiers/strings and identifies FTS5 modules. SQLite `PRAGMA table_list` supplies structural shadow types when available, with a structural FTS fallback for older supported SQLite runtimes. External/contentless FTS options keep independently owned `<fts>_content` tables authoritative; internal-content FTS bases are value-scanned while their actual shadow tables remain derived-only. `PRAGMA table_xinfo` keeps generated columns aligned with explicit projections. Raw JSON text and its parsed structure are both scanned so duplicate keys cannot erase evidence. Authoritative categories are claims, citations, events, claim embeddings, verbatim, feedback, Atlas source/evidence/action/media-retry, cache, miner, rules, entities/links, timeline, and bounded unknown. Deep, cyclic, oversized, or otherwise unscannable values are counted with `scan_incomplete` and block complete coverage without exposing their content. Artifact and spool traversal stays below resolved inputs, refuses symlinks plus Windows reparse-point junctions across supported Python versions, verifies the opened handle against contained path identity before reading, caps iterator fetches, entries, and bytes per file, scans one bounded snapshot rather than fixed overlap windows, and marks entry-limit, changed, unscannable, oversized, or truncated coverage `BLOCKED` rather than complete.

Qdrant is payload-only through an explicit verified injected scroll transport using a validated bounded limit/page count, `with_payload=true`, `with_vector=false`, exact JSON payload values, and bounded JSON offsets (including mapping/list offsets). It accepts the scroll envelope `{"result":{"points":[],"next_page_offset":...}}`, preserves partial aggregate counts, and blocks missing payloads plus repeated, deep, cyclic, hostile, or malformed offsets/payloads, oversized pages, malformed envelopes/points, page limits, invalid bounds, and transport failures with fixed `BLOCKED-EXTERNAL` reasons. Without that transport it returns `BLOCKED-EXTERNAL/qdrant_not_configured`; it does not default to localhost, probe, or discover. Invalid CLI arguments produce only fixed aggregate JSON and never echo supplied values. Any missing/unreadable artifact or spool root (including an omitted root list) is local `BLOCKED`; local SQLite/artifact/spool incompleteness takes top-level precedence over Qdrant blockers. Refused nested symlink/junction entries are aggregate evidence but do not block an otherwise readable root; a refused root or detected identity race blocks completion. A real Qdrant dry-run requires evidence of scoped, authenticated transport. Backup approval and explicit authority are required only before any later cleanup/redaction/rebuild action; this package implements none of those actions.
